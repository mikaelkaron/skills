<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workflow_dispatch inputs → setup job | Boolean inputs from GitHub UI; type-safe by GitHub Actions schema (only true/false) |
| setup job output → release job env | SEMREL_SKIP_STEPS string passes through job output — no external untrusted input beyond workflow inputs |
| release job → npm registry | NPM_TOKEN scoped secret; not exposed to ci.yml |
| release job → GitHub API | GITHUB_TOKEN auto-scoped to repo; permissions declared explicitly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Elevation of privilege | workflow_dispatch from arbitrary branch | mitigate | Branch guard `if:` on release job (D-09): only main, pre, alpha, beta, rc* allowed — non-configured branches get no-op per D-09 |
| T-02-01-02 | Tampering | Concurrent release dispatch | mitigate | concurrency.cancel-in-progress: false + group: release-${{ github.ref }} serializes dispatches per branch; in-flight release cannot be cancelled mid-run (D-11) |
| T-02-01-03 | Information disclosure | NPM_TOKEN in release job | mitigate | NPM_TOKEN declared only as env var on the npx semantic-release step; not set at workflow level or in setup job; not present in ci.yml |
| T-02-01-04 | Tampering | Dot metacharacter in @semantic-release/npm:. step ID | mitigate | Bash array uses "@semantic-release/npm:\\." — produces @semantic-release/npm:\. in SEMREL_SKIP_STEPS; regex matches only literal dot, not all npm steps |
| T-02-01-05 | Denial of service | Missing fetch-depth: 0 | mitigate | fetch-depth: 0 on checkout step (D-14); without it semantic-release would calculate incorrect version from shallow history |
| T-02-01-SC | Tampering | actions/checkout@v5, actions/setup-node@v4 | accept | Actions pinned to exact semver tags already in use in this repo (publish-tile.yml, test.yml) — verified [VERIFIED: codebase] |
</threat_model>
