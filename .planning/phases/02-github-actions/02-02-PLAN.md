<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| push/pull_request event → ci job | Untrusted code from PRs runs lint/build/test; no secrets beyond GITHUB_TOKEN |
| ci job → GitHub API | GITHUB_TOKEN used only for lcov PR comment; scoped to pull-requests:write, contents:read |
| NPM_TOKEN | Absent from ci.yml — not needed for CI; only in release.yml |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Information disclosure | NPM_TOKEN exposure in CI | mitigate | NPM_TOKEN is not referenced anywhere in ci.yml; only GITHUB_TOKEN is used (for lcov comment); PR forks cannot exfiltrate NPM_TOKEN |
| T-02-02-02 | Elevation of privilege | [skip ci] commit bypass | accept | [skip ci] check is on the ci job condition; a PR author cannot gain elevated permissions by including [skip ci] — they only skip lint/build/test on their own PR, which is a dev-convenience pattern, not a security boundary |
| T-02-02-03 | Tampering | Supply chain via romeovs/lcov-reporter-action | accept | Action pinned to exact tag @v0.4.0 (verified in existing test.yml); runs after tests complete; reads only lcov artifact; no write access beyond pull-requests:write |
| T-02-02-04 | Denial of service | Duplicate CI runs (test.yml + ci.yml) | mitigate | test.yml deleted (D-01) — deleting the file is part of this task; only ci.yml remains |
| T-02-02-SC | Tampering | actions/checkout@v5, actions/setup-node@v4 | accept | Pinned to exact semver tags already in use in this repo; no new actions introduced |
</threat_model>
